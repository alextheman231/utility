This is a new minor release of the `@alextheman/utility` package. It introduces new features and other backwards-compatible changes that should require little to no refactoring. Please read the description of changes below.
<!-- alex-c-line-release-summary-end -->

## Description of Changes

<!-- user-editable-section-start -->
- Move the `encryptWithKey` function to the `@alextheman/utility/node` entrypoint.
    - That function uses `libsodium-wrappers` internally, which causes build warnings in non-Node runtimes (e.g. React projects) if present in the main entrypoint.
    - As such, this needs to be moved to the Node entrypoint to prevent it from being used in runtimes that don't use Node.
    - Note that this is a breaking change if you're using this function! It's a quick fix hence why this is just a minor change, but worth knowing anyway.


## Migration Notes

- `import { encryptWithKey } from "@alextheman/utility";` -> `import { encryptWithKey } from "@alextheman/utility/node";`

## Additional Notes

- I need to release this now before the next major in order to unblock Lexicon work.
- Or, well, in fairness, it's not **blocked** necessarily, but I am getting a giant burst of minified content in the terminal when building the front-end, which is bothering me. Hence this somewhat urgent breaking fix.
