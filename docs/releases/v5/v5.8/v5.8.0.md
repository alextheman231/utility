## Description of Changes

<!-- user-editable-section-start -->
- Add an escapeRegexPattern function
    - This helps to ensure that any raw Regex strings we pass into RegExp gets properly escaped before we use it.
    - It prevents unexpected behaviour when working with dynamic patterns (e.g. paths, user input, or configuration values).
    - It will be particularly useful when constructing regex patterns from values such as router base paths, where special characters may otherwise break matching.
