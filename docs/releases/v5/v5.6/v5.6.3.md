## Description of Changes

<!-- user-editable-section-start -->
- Release the feature docs to GitHub Pages
    - Now on release of this package, instead of deploying the feature docs to Netlify, we instead will be deploying to GitHub Pages.
    - This feels cleaner and keeps the feature docs more closely-related to this repository.
    - Netlify is generally better for larger-scale applications, whereas GitHub Pages is better for package documentation, from what I understand.

## Additional Notes

- Note that the extra workflow is hardcoded into the repository as of now. If it works here, I will eventually abstract it out into the github-actions repository.
- But for now, since this is pretty much the only repository with hosted documentation, I figured it'd be best to just use it here first directly to save us from having to potentially do a whole bunch of patch releases afterwards if it all goes wrong.
