This is a new patch release of the `@alextheman/utility` package. It fixes issues with the package and should require no refactoring. Please read the description of changes below.
<!-- alex-c-line-release-summary-end -->

## Description of Changes

<!-- user-editable-section-start -->
- Throw a `DataError` at runtime for inputs to `VersionNumber` constructor that are not valid at compile-time by the type system
    - This should be helpful in case we need to parse version numbers from external sources, such as an API request, where responses may be typed as any and provide a type that our type system shouldn't really allow.
    - In practice it should be rare to have to bypass the type system like that, but either way we do now guard against this natively in `VersionNumber` itself.
- Make the radix less than 10 character checks more rigourous
    - The validation now strips any leading sign `(+ or -)` before checking digits against the radix.
    - This avoids relying on parseInt(character) returning `NaN` for non-digit characters, which could otherwise lead to ambiguous comparisons.
