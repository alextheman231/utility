## Description of Changes

<!-- user-editable-section-start -->
- Bring the TypeDoc configuration to this repository directly.
    - Previously it was being brought in from my ESLint plugin.
    - This was problematic because it meant that our own TypeDoc usage is tightly coupled to the plugin, even though this repository is the only one using that config.
    - Furthermore, it adds more surface area to the plugin, where it becomes responsible for things it doesn't need to be responsible for.

## Additional Notes

- This release note exists purely to test that after bringing the TypeDoc configuration here, the full publish workflow still works.
- The main cause of failure for such a refactor (that is, making the `publish-github-pages` workflow config-build aware) has already been addressed, so hopefully it will succeed first try, right?
