This is a new patch release of the `@alextheman/utility` package. It includes small, non-breaking changes and should require no refactoring. Please read the description of changes below.
<!-- alex-c-line-release-summary-end -->

## Description of Changes

<!-- user-editable-section-start -->
- Update tsdown and don't bundle TypeScript in config builds.
    - Previously the config builds failed because it tried to include TypeScript in the bundle, which caused issues with type imports from TypeScript being unrecognised.
    - This change excludes it from the bundle, therefore ensuring that tsdown does not try including the TypeScript types in the bundle, allowing the build to pass.
- Add Zod as a peer dependency.
    - We were getting quite a few type errors (including even a few memory failures with tsc) due to this package bringing in its own version of Zod on installation.
    - This opens up the possibility of mismatches between Zod versions, which can cause issues in cases where the types between versions may be subtly different.
    - This fixes that by making Zod a peer dependency of the package, therefore allowing the Zod context to be shared between consuming apps and this package.
