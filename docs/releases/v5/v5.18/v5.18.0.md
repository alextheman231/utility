This is a new minor release of the `@alextheman/utility` package. It introduces new features and other backwards-compatible changes that should require little to no refactoring. Please read the description of changes below.
<!-- alex-c-line-release-summary-end -->

## Description of Changes

<!-- user-editable-section-start -->
- Allow the `fillArray` function to run Promises sequentially if the `sequential` option is given.
    - It will default to running in parallel as was the default behaviour before.
    - However, now if `{ sequential: true }` is given as an option, it will resolve each item one after the other rather than all at the same time.
    - This is particularly helpful in cases where, for example, you want to fill an array with results of a database query using the same connection, but you don't want to run parallel queries under the same connection as that may cause conflicts.
